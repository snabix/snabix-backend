# Changelog

Все важные изменения в этом проекте документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
а сам проект следует принципам [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.3.0] - 2026-05-07

### Added
- Расширен backend-контур авторизации и управления аккаунтом
  - `POST /api/v1/auth/forgot-password`
  - `POST /api/v1/auth/reset-password`
  - `PATCH /api/v1/auth/me`
- Добавлена доменная и прикладная логика
  - запроса на сброс пароля
  - завершения сброса пароля
  - обновления профиля текущего пользователя
- Добавлено письмо для сброса пароля и queued job для его отправки
- Добавлены отдельные feature-тесты для всех актуальных auth API-методов
  - `sign-up`
  - `sign-in`
  - `verify-email`
  - `me`
  - `update-profile`
  - `logout`
  - `forgot-password`
  - `reset-password`
- Добавлена системная модель логирования ключевых действий пользователей
  - таблица `system_logs`
  - Filament resources для `system logs` и `error logs`
  - listener для сохранения loggable events
- Добавлены доменные события для ключевых auth-сценариев
  - успешный вход
  - неуспешная аутентификация
  - регистрация
  - запрос подтверждения email
  - подтверждение email
  - выход из системы
  - обновление профиля
  - запрос сброса пароля
  - завершение сброса пароля
- Добавлены backend-настройки для frontend redirect URL
  - восстановление пароля
  - завершение email verification

### Changed
- Логирование ключевых действий переведено с прямых вызовов в handler'ах на событийную модель через `events + listener`
- Страница состояния системы в Filament доработана до более полезного dashboard-режима
  - добавлены метрики по памяти и диску
  - добавлены overview/check widgets
  - улучшено отображение состояния инфраструктуры
- Swagger / OpenAPI-документация расширена под новые auth API-методы
- Локализация backend-интерфейса дополнена
  - обновлены переводы `ru`
  - типовые Filament labels для распространённых полей переведены на `translateLabel()`

### Fixed
- Исправлена отправка queued email после регистрации пользователя
  - восстановлена корректная цепочка `event -> listener -> job`
- Исправлены сценарии тестов auth API и тестовая инфраструктура под текущую доменную модель
- Исправлены отображение и вычисление системных resource-метрик для health checks
- Исправлены подписи типовых полей в Filament resources, чтобы они брались из переводов единообразно

## [0.2.0] - 2026-05-06

### Added
- Реализован backend-контур авторизации пользователей
  - `POST /api/v1/auth/sign-up`
  - `POST /api/v1/auth/sign-in`
  - `GET /api/v1/auth/verify-email`
  - `GET /api/v1/auth/me`
  - `POST /api/v1/auth/logout`
- Добавлена доменная и прикладная логика регистрации, входа, выхода, получения профиля и подтверждения email
- Добавлена отправка письма для подтверждения email через queued job
- Добавлен endpoint профиля текущего пользователя по Sanctum token
- Добавлены тесты, покрывающие текущие API-методы авторизации
- Добавлена OpenAPI / Swagger-документация для auth API
  - описания request/response схем
  - описание bearer security scheme
  - генерация спецификации для Swagger UI
- Добавлена страница состояния системы в Filament admin panel

### Changed
- Разделены обычные пользователи и администраторы на backend-уровне
  - пользователи продолжают работать через сущность `users`
  - администраторы вынесены в отдельную модель и таблицу `admins`
  - админ-панель Filament переведена на guard `admin`
  - в `/admin` добавлено раздельное отображение пользователей и администраторов
- Очереди переведены с `database queue` на `RabbitMQ`
  - добавлен Laravel RabbitMQ driver
  - обновлена конфигурация `queue.php`
  - добавлены переменные окружения для RabbitMQ
  - добавлен отдельный `queue-worker` сервис в Docker
  - `composer` dev workflow переведён на `queue:work rabbitmq`
- Backend-конфигурация приведена к работе с Redis и RabbitMQ в контейнерной среде
- Конфигурация `healthcheck` адаптирована под текущую инфраструктуру backend

### Fixed
- Исправлена цепочка регистрации пользователя, из-за которой event/listener могли не запускаться
- Исправлена запись `email_verified_at` после перехода по signed verify-email ссылке
- Исправлены DI bindings, необходимые для корректного выполнения email job внутри worker
- Устранены проблемы тестового слоя
  - добавлена корректная привязка factory к модели пользователя
  - устранены падения тестов на нестандартной доменной валидации имени
- Исправлена Swagger/OpenAPI-конфигурация, из-за которой генерация документации падала без корневого `Info`

## [0.1.0] - 2026-05-02

### Added
- Добавлена базовая локальная инфраструктура для разработки на Docker
  - `php-fpm` контейнер приложения
  - `caddy` как входной HTTP-сервер
  - `postgres` как основная база данных
- Добавлена административная панель на Filament
- Добавлена CLI-команда создания администратора `app:make-admin`
- Добавлена PostgreSQL- и Sanctum-ориентированная конфигурация приложения для локальной среды
- Добавлены developer-команды в `Taskfile.yaml`
  - управление Docker-контейнерами
  - запуск `php-cs-fixer`
  - вход в контейнер приложения

### Changed
- Локальная среда проекта переведена с дефолтной Laravel-конфигурации на Docker-first запуск через `Caddy + PHP-FPM + PostgreSQL`
- Конфигурация окружения и `docker-compose` приведены к схеме, где основные значения берутся из `.env`
- Усилен и расширен конфиг `php-cs-fixer`
  - добавлены более строгие правила форматирования
  - расширено покрытие основных Laravel-папок и bootstrap/public entrypoints

### Fixed
- Исправлена схема `Taskfile`, из-за которой задачи с `php-cs-fixer` не проходили валидацию
- Устранён рассинхрон между локальной PostgreSQL-конфигурацией и Docker-окружением
- Стабилизирована доступность Filament admin panel в контейнерной среде через `Caddy`

### Added (Planned)
- Разделение обычных пользователей и администраторов
- Добавление прав доступа в административной панели
- Полная авторизация пользователей
- Настройка git hooks
- Добавление `Larastan / PHPStan` для более строгой проверки кодовой базы
- Единый pre-push quality gate
  - проверка кодовой базы
  - проверка стилистики кода
  - выполнение всех тестов перед отправкой изменений в удалённый репозиторий
