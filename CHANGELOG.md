# Changelog

Все важные изменения в этом проекте документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
а сам проект следует принципам [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.6.2] - 2026-05-18

### Changed
- Правила пользовательской публикации объявлений частично вынесены из `EloquentListingRepository` в доменную `ListingPublicationPolicy`
  - обычное создание объявления переводит запись в статус `pending_review`
  - явное сохранение черновика оставляет статус `draft`
  - обязательные характеристики проверяются для не-черновиков
- Репозиторий объявлений стал получать готовое решение о проверке required-характеристик и больше не определяет это правило по статусу самостоятельно

### Added
- Добавлены тесты для `ListingPublicationPolicy`
- Добавлен feature-тест, фиксирующий обязательность required-характеристик при обновлении не-чернового объявления

## [0.6.1] - 2026-05-17

### Added
- Добавлены feature-тесты для пользовательских сценариев объявлений
  - защита модерационных полей при создании объявления
  - сохранение модерационного состояния при обновлении объявления
  - создание объявления в статусе `pending_review` при обычной отправке
  - сохранение объявления как черновика через явный флаг `saveAsDraft`
  - обязательность заполнения required-характеристик категории
  - возможность сохранить черновик без обязательных характеристик
  - очистка пустых и больше не применимых значений характеристик
  - отсутствие приватных owner/contact-полей в публичном списке объявлений
- Добавлен mapper публичной полезной нагрузки объявлений для безопасной отдачи данных наружу
- Добавлен отдельный demo-сидер каталога категорий и характеристик категорий
  - базовые товарные категории
  - базовые сервисные категории
  - подготовленные характеристики для разных типов объявлений

### Changed
- Пользовательское создание объявления теперь переводит запись в статус `pending_review`, если пользователь не сохраняет объявление как черновик
- Статус `draft` теперь используется только для явного сценария сохранения черновика
- Пользовательское обновление объявления больше не изменяет модерационные поля
  - `status`
  - `is_featured`
  - `rejection_reason`
- Синхронизация значений характеристик объявления стала строже и предсказуемее
  - required-характеристики проверяются независимо от наличия входного массива
  - пустые optional-значения удаляются
  - характеристики, больше не относящиеся к выбранной категории, очищаются
- Контроллеры объявлений дополнительно разгружены от request-маппинга через явные методы `FormRequest`
- Filament-форма характеристик категорий стала спокойнее для администрирования
  - выбор типа характеристики заменен на `select`
  - тексты помощи переписаны на русском языке
  - кнопка создания характеристики приведена к стандартной подписи `Создать`
  - страницы ресурса характеристик получили явные заголовки и хлебные крошки
- Глобально отключена кнопка `Создать и создать еще` в Filament create-страницах

### Fixed
- Публичный список объявлений больше не раскрывает приватные данные владельца и контактов
- Устранены ошибки статического анализа в модуле объявлений, характеристик категорий и Filament-форме объявлений
- Добавлены недостающие переводы Filament-label для ресурса характеристик категорий

## [0.6.0] - 2026-05-13

### Added
- Добавлен базовый доменный фундамент для модуля объявлений
  - схема `listings`
  - схема `category_attribute_definitions`
  - схема `listing_attribute_values`
  - enum-типизация для вида объявления, статуса и состояния
- Добавлен Filament resource для характеристик категорий
  - создание
  - редактирование
  - просмотр
  - удобные формы для подготовки готовых полей объявлений
- Добавлен прикладной и HTTP-слой для черновиков объявлений
  - `POST /api/v1/listings`
  - `GET /api/v1/listings/{listingId}`
  - `PATCH /api/v1/listings/{listingId}`
- Добавлен endpoint характеристик категории с учетом наследования
  - `GET /api/v1/categories/{categoryId}/attributes`
- Добавлено разделение каталога категорий по направлениям
  - `product`
  - `service`
- Добавлены operational-команды и расписание обслуживания backend
  - очистка `system_logs`
  - очистка `failed_jobs`
  - очистка устаревших reset-токенов
  - live-просмотр очередей и worker-логов через `Taskfile`
- Добавлен `dedoc/scramble` для автоматической генерации API-документации
  - `GET /docs/api`
  - `GET /docs/api.json`
  - task-команды для анализа и экспорта документации
- Добавлена единая команда полной backend-проверки
  - `task check`
  - `composer check`

### Changed
- Проект отказался от `l5-swagger` и ручных `OpenAPI`-атрибутов в HTTP-слое
  - единственным источником API-документации стал `Scramble`
- Новый модуль объявлений переведен на `int-backed enum`
  - `catalog_type`
  - `listing.type`
  - `listing.status`
  - `listing.condition`
  - `category_attribute_definitions.type`
- API категорий, характеристик и объявлений теперь отдает числовые enum-значения вместе с человекочитаемыми label-полями
- Политика backend-логирования стала более практичной
  - в БД больше не пишется шум от большинства безопасных HTTP-запросов
  - сохранены ошибки, небезопасные методы и медленные запросы
- Стратегия очередей выровнена между `docker-compose` и `Taskfile`
  - введён единый `QUEUE_WORK_QUEUES`
- Системный логгер получил fallback в `stderr`, если DB-персистентность логов недоступна
- API категорий подготовлен к дальнейшему разделению веток товаров и услуг через `catalogType`
- API категорий и объявлений приведен к более стабильному контракту
  - корневые категории документированы через актуальный `/api/v1/categories/list`
  - добавлен единый mapper полезной нагрузки объявления для `create / show / update`
- Контроллеры объявлений очищены от прямой выборки сырых request-полей
  - чтение данных вынесено в явные accessor-методы `FormRequest`
- Новые миграции приведены к единому стилю
  - временные поля перенесены в начало таблиц
- География временно убрана из схемы объявлений
  - `region`
  - `city`
- Модель характеристик оформлена как подготовка готовых полей формы объявления
  - администратор создает свойства для категории
  - пользователь только заполняет уже подготовленные поля при создании объявления
- Модульные backend-провайдеры расширены под новые bounded contexts
  - `CatalogServiceProvider`
  - `ListingServiceProvider`

### Fixed
- Исправлен operational-рассинхрон, из-за которого worker в Docker и ручной worker могли слушать разные очереди
- Удалён устаревший backend-конфиг для redirect-based email verification
- Уточнены `.env` и maintenance-настройки очередей под текущую cookie-auth и queue-архитектуру
- Обновлён тестовый и quality-check контур backend
  - удалены устаревшие Swagger-specific тесты
  - добавлены проверки доступности `Scramble`-документации
  - введена единая команда `task check` для `cs-fixer`, `phpstan`, `Scramble` и `php artisan test`
